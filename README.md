# say-draw —— AI 语音绘图工具

纯语音控制的绘图工具:用户不使用鼠标/键盘绘图,仅通过语音指令完成绘图创作(允许点击"按住说话"按钮)。

> 七牛云比赛 · 命题二:AI 语音绘图工具

## 功能(规划)

- **L1 图元绘制**:圆 / 矩形 / 线 / 三角 / 箭头 / 文字 + 颜色、大小、九宫格位置
- **L2 对象编辑**:选中、移动、改色、缩放、旋转、删除(编号角标 + 焦点指代消解)
- **L3 画布操作**:撤销 / 重做 / 清空、导出
- **L4 复杂场景**(冲刺):语音描述 → 生成可继续编辑的具名图元组
- **L5 相对定位 / L6 语音(TTS)反馈**(冲刺)

详见 [设计文档](docs/design.md)。

## 技术栈

- 前端:Vite + TypeScript + Konva
- 语音输入:Web Speech API(zh-CN);语音反馈:SpeechSynthesis
- 指令理解:规则解析(快路) + 豆包大模型(慢路,兜底口语化 / 复杂指令,结构化 JSON 输出);L4 复杂场景文生图用 Seedream
- 后端:薄代理(转发火山方舟 Ark,密钥不进浏览器)

## 运行方式(从零复现)

> 适用于**全新设备,从零开始**。当前可体验:**语音绘制基础图元**(圆 / 方块 / 三角 / 线 + 颜色 / 大小 / 九宫格位置)。后端豆包语义解析为可选步骤(见 §5)。

### 1. 环境要求

- **Node.js ≥ 18**(含 npm):https://nodejs.org ,或 macOS `brew install node`
- **Git**
- **浏览器:Chrome 或 Edge**(语音识别依赖 Web Speech API;Safari / Firefox 不保证)
- 一个可用的**麦克风**

### 2. 获取代码

```bash
git clone https://github.com/xzha0637-design/say-draw.git
cd say-draw
```

### 3. 启动前端

```bash
cd frontend
npm install        # 安装依赖(首次必跑)
npm run dev        # 开发服务器,默认 http://localhost:5173 并自动打开
```

> 若自动打开的不是 Chrome/Edge,请手动用 **Chrome / Edge** 打开 http://localhost:5173 。

### 4. 使用 / 测试

1. 点击页面底部「🎤 开始聆听」,在浏览器弹窗中**允许麦克风权限**。
2. 说出绘图指令,例如:
   - "画一个红色的圆"
   - "在左上角画一个蓝色的方块"
   - "画一个大的绿色三角形"
   - "清空"(清空画布)
   - "撤销"(撤回上一步)/ "重做"(恢复被撤销的一步)
   - "删掉 2 号"(按右上角编号删除某个图形;支持「第三个」「十号」等中文说法)
   - "把 2 号变红 / 变大 / 移到左上角"(按编号修改某个图形的颜色、大小、位置,可组合)
   - "把它变大" / "这个删掉" / "上一个移到右边"(用「它 / 这个 / 上一个」指代**最近操作的图形**,不必报编号)
   - "帮我在右下角整个小小的橙色三角形呗"(口语化 → 自动走豆包慢路,需先启动后端 §5)
3. 对应图形出现在画布上,**每个图形右上角带编号角标(1、2、3…)**——这是后续「删掉 2 号」「把 3 号变大」等按编号编辑能力的地基;底部字幕显示识别结果与执行反馈。
4. 点「⏹ 停止聆听」结束;关闭服务器在终端按 `Ctrl+C`。

> **双路已打通**:规则快路即时处理常见指令;**规则解析不出的复杂 / 口语化指令会自动转后端豆包慢路**(需同时启动后端,见 §5;后端未启动则仅快路可用,不影响基础绘图)。

> **语音反馈(TTS)**:每条指令执行后会朗读结果(如「已画 大号圆(中间)」「已删除 2 号」);播报期间自动静麦,避免把自己的反馈当成新指令(防回环)。无需配置,Chrome / Edge 自带。

### 5.(可选)后端:豆包语义解析(慢路)

> 前端规则快路即可画基础图元;**复杂 / 口语化指令会自动转后端豆包(双路已打通)**。启动后端后,前端说「帮我整个圆呗」这类口语指令也能画;也可单独用 `curl` 验证后端。后端未启动不影响基础绘图。

**5.1 准备火山方舟密钥与模型**:火山引擎控制台 → 方舟(Ark) → 在「开通管理」**开通一个豆包对话模型**(推荐 `doubao-seed-2.0-mini` 或 `doubao-1.5-lite-32k`;注意要**对话/语言**模型,不是 Seedream 文生图) → 「API Key 管理」创建密钥。模型 ID 即填入 `ARK_MODEL`。

**5.2 配置环境变量**:

```bash
cd backend
cp .env.example .env       # 然后编辑 .env,填入 ARK_API_KEY 与 ARK_MODEL
                           # (ARK_THINKING 已预设 disabled:推理模型关思考、解析提速 2~3 倍)
```

**5.3 安装并启动**:

```bash
npm install
npm run dev                # 默认 http://localhost:8787
```

**5.4 验证(另开一个终端)**:

```bash
curl http://localhost:8787/api/health
# 预期:{"ok":true}

curl -X POST http://localhost:8787/api/parse \
  -H 'Content-Type: application/json' \
  -d '{"text":"画一个天蓝色的大圆"}'
# 预期:{"ok":true,"command":{"action":"draw","shape":"circle","color":"#87CEEB","size":"large","position":"center"}}
```

> 未配置 `.env` 时,`/api/parse` 返回提示「后端未配置 ARK_API_KEY / ARK_MODEL」——属正常,填好密钥即可。

### 生产构建

```bash
cd frontend
npm run build      # 产物输出到 frontend/dist/
npm run preview    # 本地预览构建产物
```

## 依赖说明

> 按比赛要求列明第三方依赖并标注原创部分,随开发补全。

- **第三方**:前端 konva、vite、typescript;后端 express、dotenv(详见各模块 `package.json`)。豆包大模型经**火山方舟 OpenAI 兼容接口**调用,后端用原生 `fetch`,无需额外 SDK。
- **原创部分**:指令文法与规则解析、双路路由、后端解析提示词与归一化校验、编号+焦点编辑模型、草图先行渲染、执行引擎等。

```
/backend    后端薄代理(转发火山方舟豆包,语义解析)
/docs       设计文档
```
